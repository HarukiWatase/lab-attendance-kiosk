# フロント（キオスク）仕様書

本書は、研究室打刻用フロントを **ラズパイ＋QRスキャナ本番** と **開発環境** でどう切り替えるかを固定する。UI の大枠は現行 `frontend/src/App.tsx` に沿い、大きなデザイン変更はしない。

---

## 1. 目的と前提

- **打刻**: USB HID 相当の QR スキャナ（Tera TR-UM9200-JP 想定）で `user_id` を送る。
- **サフィックス**: **CR+LF** に設定済み（読取完了 ≒ Enter 相当）。
- **端末**: Raspberry Pi、ブラウザは Chromium 想定。マウスは補助（タブ切替など）。
- **正式な月次・教授向け指標**: スプレッドシート側（本フロントの分析は補助表示）。

---

## 2. 本番キオスク仕様

### 2.1 画面

- **打刻タブ**と**分析タブ**の両方を表示する（現行と同様のタブ UI）。
- 見た目・トーンは現状のピクセル風パネル等を**大きく変えない**。

### 2.2 本番で非表示にするもの

次は **本番ビルドでは表示しない**（誤操作・不要 UI の排除）。

- ユーザー ID の**可視テキスト入力**
- **scan** ボタン
- **mock scan** ボタン（開発専用）

### 2.3 本番で維持・必須とする挙動

- **スキャナ入力の受け口**: 画面上に入力欄は見せないが、**キーボードウェッジを確実に受けるための非表示フォーカス要素**（例: `opacity:0`・画面外配置の `input`）を置く。実装時に詳細化する。
- **確定条件**: CR+LF 前提のため、**Enter 相当**でバッファを確定し `POST /api/scan` する。必要なら将来、CR のみ端末向けフォールバックを検討（現時点では CR+LF 固定でよい）。
- **タブ切替**: マウスで **分析** を開ける（補助操作）。

### 2.4 分析タブ

- **表示する**。
- **データ取得**
  - `GET /api/view/users` … アプリ起動時に1回（打刻時の表示名用）。
  - `GET /api/view/analytics/week-calendar` … **分析タブを開いたときに毎回再取得**。
- **指標**: **JST・月曜始まりの暦週**（`session_log.week_start` と一致）における **在室合計時間（h）**。`is_auto_fixed=TRUE` のセッションは含めない（GAS側で除外）。
- **週の切替**: 新しい暦週が始まると API の `week_start` が変わるため、**先週の棒は表示されず「今週だけ」**になる。
- **進捗バー**: 週合計を **15h** を100%とみなした目安（学期判定の15h週平均とは別。画面に注記）。
- 現行どおり **60 秒で打刻タブへ戻す**動きを維持してよい（キオスクで長時間分析を置きたい場合は別途要件化）。

### 2.5 エラー・メッセージ

- クールダウン・未登録・通信エラーは、現行の子ども向けひらがなメッセージ方針を踏襲。
- **request_id** の画面表示は任意（運用で必要になれば追加）。

---

## 3. 開発環境仕様

- **可視入力**・**scan**・**mock scan** を**表示**する（デバッグ・PC 操作向け）。
- `import.meta.env.DEV` によりデモ用フォールバックの挙動が残る場合がある（現行 `App.tsx` の方針）。

---

## 4. 本番／開発の切り替え（実装方針）

- **推奨**: `import.meta.env.PROD`（または `import.meta.env.MODE === "production"`）で UI 部品の出し分けを行う。
- **代替**: `VITE_KIOSK=1` のような環境変数で明示的にキオスクモードにする（デプロイ手順と合わせて1つに統一する）。

※ リポジトリへの反映時は、上記のどちらかに統一し本書に1行追記する。

**実装:** `import.meta.env.DEV` で開発 UI を出し分け（`vite build` の本番バンドルでは非表示）。

---

## 5. API・プロキシ

- フロントは相対パス `/api/...` を前提（Vite dev のプロキシ or 本番リバースプロキシでバックエンドへ転送）。
- **ラズパイ本番の nginx・systemd・Chromium 例:** `deploy/raspberry-pi/README.md`
- **分析（暦週）**: `GET /api/view/analytics/week-calendar` … 応答 `{ week_start, items: [{ user_id, display_name, week_total_hours }] }`。
- （任意・レガシー）`GET /api/view/analytics/semester` … スプシ `summary_semester` 由来。フロントの分析タブでは使用しない。
- ラズパイ本番の具体構成は `deploy/raspberry-pi/` を参照。
