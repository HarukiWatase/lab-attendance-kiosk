# 勤怠管理システム 運用ディスプレイ向けUI仕様書

本仕様書は、通常サイズのPCディスプレイでの常設運用を前提に、画面構成・表示ルール・操作ルールを定義する。  
対象は `frontend` のUI実装であり、打刻運用の安定性と可読性を最優先とする。

---

## 1. 固定要件（確定）

- 既定表示は **打刻タブ固定**
- 状態メッセージのフォントサイズは **推奨サイズ**を採用
- 直近ログは **3件のみ表示**
- 分析タブは以下の2要素に限定
  - チーム週平均
  - ユーザー別進捗バー
- ユーザー数は **14名程度** を想定
- 通常運用の入力デバイスは **QRスキャナーのみ**
- スキャンボタン押下は通常不要（自動送信）

---

## 2. 画面構成

## 2-1. タブ構成

- タブは2つ:
  - `打刻`（初期表示）
  - `分析`
- 起動時・リロード時は必ず `打刻` を表示する

## 2-2. 打刻タブレイアウト

1. ヘッダ
   - タイトル: `勤怠管理システム`
2. 操作領域
   - `user_id` 入力欄
   - `scan` ボタン（非常時の手動入力用）
   - （開発時のみ）`mock scan` ボタン
3. 状態メッセージウィンドウ
   - `idle / processing / success_in / success_out / blocked / error`
4. 直近ログ領域
   - 表示件数は最新3件
   - 各行: `時刻 / user_id / result(必要に応じてaction)`

## 2-3. 分析タブレイアウト

1. チーム週平均カード
   - `xx.xh / 15h`
   - 進捗バー（0〜100%）
2. ユーザー別進捗一覧（14名想定）
   - 1行ごとに:
     - 氏名（または user_id）
     - `xx.xh`
     - ステータス（達成/注意/要改善）
     - 水平バー
   - 縦スクロール対応（14名が1画面に収まらない場合を許容）

---

## 3. タイポグラフィ・サイズ仕様（推奨）

運用時の遠目視認性を確保するため、以下を推奨値として固定する。

- タイトル: `32px`
- タブラベル: `20px`
- 状態メッセージ: `28px`（最重要）
- 入力欄/ボタン文字: `20px`
- 直近ログ文字: `16px`
- 分析タブ行文字: `16px`

最低基準:
- 重要情報（状態メッセージ）は `24px` 未満にしない

---

## 4. カラー・状態表現

- `idle`: 白
- `processing`: 水色（必要なら点滅）
- `success_in/success_out`: 緑系
- `blocked`: 黄系
- `error`: 赤系

進捗バー（分析タブ）:
- 達成（`>=15h`）: 緑
- 注意（`12〜15h`）: 黄
- 要改善（`<12h`）: 赤

---

## 5. 操作仕様

- スキャナー入力完了（Enter受信）時:
  - 自動で `/api/scan` を実行
  - `processing` に遷移
- `scan` 押下時（非常時）:
  - 手動で同じ処理を実行
- 通信中:
  - `scan` / `mock scan` は無効化（連打防止）
- 成功/失敗後:
  - 3秒表示後に `idle` へ戻る
  - 入力欄へ自動再フォーカス
- 直近ログ:
  - 最新3件のみ保持・表示
  - 4件目以降は古いものを破棄
- 分析タブ:
  - 60秒無操作で `打刻` タブへ自動復帰

---

## 6. 14名表示時の分析タブ仕様

- 1行の高さ目安: `40〜48px`
- 一覧領域は `max-height` を設けて縦スクロール
- 表示順:
  - 既定は `weekly_hours` 降順（目標達成状況を見やすくする）
  - 将来、氏名順切替を追加可能

---

## 7. データ連携仕様（現段階）

- 打刻タブ: 既存API（`/api/scan`, `/api/mock-scan`）
- 分析タブ:
  - 初期はダミーデータでも可
  - 次段階で `summary_semester` 相当APIに接続

想定レスポンス（将来）:

```json
[
  {
    "user_id": "A12345",
    "display_name": "山田 太郎",
    "weekly_avg_hours": 14.2
  }
]
```

---

## 8. 受け入れ基準

- 起動時に必ず `打刻` タブが開く
- 状態メッセージが `28px` で表示される
- 直近ログが常に3件までである
- 分析タブで
  - チーム週平均カード
  - ユーザー別進捗バー（14名）
  が表示される
- 14名表示時にレイアウトが崩れない（スクロールで閲覧可能）
- スキャナー入力（Enter）だけで打刻できる
- 分析タブから60秒後に打刻タブへ戻る

---

## 9. 実装メモ（開発向け）

- タブ状態:
  - `const [tab, setTab] = useState<"scan" | "analytics">("scan")`
- 直近ログ:
  - `setLogs((prev) => [newLog, ...prev].slice(0, 3))`
- 分析一覧:
  - `overflow-y: auto` と `max-height` を設定
- フォント:
  - DotGothic16維持
